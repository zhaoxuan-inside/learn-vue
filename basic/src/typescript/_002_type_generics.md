# 断言

TypeScript 的类型断言（Type Assertion）在“告诉编译器如何处理类型”这个意义上，确实类似于其他语言中的强制类型转换。但它们在底层机制和安全性上有本质区别

```typescript
human as Human
```

| 特性         | TypeScript 类型断言        | Java/C# 强制类型转换                     |
| ------------ | -------------------------- | ---------------------------------------- |
| 作用时间     | 仅在编译时                 | 运行时也会执行                           |
| 实际改变数据 | ❌ 不改变，纯粹是类型提示  | ✅ 可能转换数据表示（如 (int) 3.14 → 3） |
| 运行时检查   | ❌ 不做任何检查            | ✅ 会检查类型兼容性，失败时抛异常        |
| 性能影响     | 无（编译后断言代码被移除） | 有（需要执行转换逻辑）                   |
| 安全性       | 由开发者保证，不安全       | 语言提供一定安全保障                     |
