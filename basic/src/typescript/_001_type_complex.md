## interface 和 class 的区别

| 维度         | class                         | interface                        |
| ------------ | ----------------------------- | -------------------------------- |
| 本质         | 代码实现（值 + 类型）         | 纯类型约束（编译后消失）         |
| 运行时存在   | ✅ 存在（生成 JS 代码）       | ❌ 不存在（编译后完全删除）      |
| 能否实例化   | ✅ 可以 new ClassName()       | ❌ 不可以                        |
| 能否包含实现 | ✅ 可以（方法体、默认值等）   | ❌ 只能描述形状，无实现          |
| 访问修饰符   | ✅ private、protected、public | ❌ 不支持（仅描述，无访问控制）  |
| 可扩展性     | 单继承（extends 一个类）      | 多继承（可 extends 多个接口）    |
| 可声明合并   | ❌ 不允许重复定义             | ✅ 同名接口自动合并              |
| 主要用途     | 创建对象、封装逻辑、继承行为  | 定义契约、约束对象形状、解耦设计 |
